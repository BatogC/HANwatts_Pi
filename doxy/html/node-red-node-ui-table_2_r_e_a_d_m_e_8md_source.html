1 node-red-node-ui-table
2 ======================
3 
4 A Node-RED UI widget node which displays data as a table.
5 
6 ## Install
7 
8 Either use the Editor - Menu - Manage Palette - Install option, or run the following command in your Node-RED user directory (typically `~/.node-red`) after installing Node-RED-dashboard.
9 
10  npm i node-red-node-ui-table
11 
12 ## Usage
13 
14 This table node expects `msg.payload` to contain an array of data, one object per row.
15 Each data row object should have the same set of keys because the keys in the object are used as the column names.
16 
17 Both examples can be imported from the Node-RED Editor - Menu - Import - Examples
18 
19 ### Simple Table
20 
21 With no configuration the node will try to create a table with equally spaced columns of simple text for each row provided, using the keys as column titles.
22 
23 ![screenshot](https://raw.githubusercontent.com/node-red/node-red-ui-nodes/master/node-red-node-ui-table/screenshot.png)
24 
25 ### Richer Table
26 
27 The columns can be configured manually. If so then only the `msg.payload` properties defined will be displayed. You can then also define the Title, Width, Alignment and Format of the column.
28 
29 ![screenshot2](https://raw.githubusercontent.com/node-red/node-red-ui-nodes/master/node-red-node-ui-table/screenshot2.png)
30 
31  - **Title**: Text for the column title (or blank).
32  - **Width**: Either a number of pixels or percentage of the overall table width. e.g. 150 or 20%. Leave blank for automatic, equally spaced to fill the available space.
33  - **Align**: Column alignment, left, centre or right.
34  - **Format**: Formatting of the input.
35  - **Plain Text** - Simple text values.
36  - **HTML** - Rich html to allow text Formatting - *NOTE*: this is raw un-sanitised HTML.
37  - **Link** - URL link to a web page.
38  - **Image** - Source (src) URL of an image to display.
39  - **Progress** - a progress bar from 0 to 100.
40  - **Traffic** - Red/Amber/Green indicator light set by numbers in the range 0-33-67-100.
41  - **Color** - HTML color name, or hex value (#rrggbb) to fill the cell.
42  - **Tick/Cross** - Tick or Cross symbol, boolean true/false, numeric 1/0 or text "1"/"0".
43  - **Stars** - Number of stars - numeric 0 to 5.
44  - **Row Number** - Current row number.

65 ## advanced features
66 
67 ui-table is based on the **tabulator** module. You can find an excellent in depth [documentation here](http://tabulator.info/docs/4.4) with many [examples here](http://tabulator.info/examples/4.4).
68 
69 ## send commands to ui-table
70 
71 Instead of sending an array to ui-table this node to replace the complete table data ui-table also accepts an object as payload to send commands. Beside data manipulation you can [set filters](http://tabulator.info/docs/4.5/filter#func) and do many other things with commands. The object must have the following properties
72 
73 - `command` a valid tabulator function such as `addRow`, `replaceData` or `addFilter`
74 - `arguments` *(optional)* array of arguments for that function
75 - `returnPromise` *(optional)* a boolean value. `true` if the function should return a promise message. See tabulator documentation which commands will return promises
76 
77 example
78 ```json
79 {"payload":{
80  "command":"addData",
81  "arguments":[
82  {
83  "facility":"daemon",
84  "facilityCode":3,
85  "severity":"info",
86  "severityCode":6,
87  "tag":"systemd[1]",
88  "timestamp":"2020-01-02T19:17:39.793Z",
89  "hostname":"localhost",
90  "address":"127.0.0.1",
91  "family":"IPv4",
92  "port":38514,
93  "size":80,
94  "msg":"some demo data",
95  "id":2351
96  },
97  true
98  ],
99  "returnPromise":true
100  }
101 }
102 ```
103 By sending only changed or new data to ui-table it is possible to update the table very fast by only sending the new data down to cell level. Or huge amounts of data could be sent like logs.
104 
105 **important notices**
106 
107 Data which is sent to ui-table through commands is **not** cached by ui-table! The flow has to take care to update the table for new clients connection or dashboard tab changes!
108 Tabulator does not limit the amount of data it holds. It is quite efficient in showing tables with a couple of thousand rows. If it the data exceeds the capabilities of the clients browser it will crash with an **out of memory** error without notice.
109 
110 Example flow "4 sending commands.json" file can be found in the examples folder or installed directly using **menu/import/examples/ui-table**.
111 This flow shows a basic implementation how the flow can keep a cached copy of all table data and add/delete or update selective rows.
112 Most nodes have info text available in the info/help tab.

114 ## control ui-table by sending ```msg.ui_control``` messages
115 
116 ui-table can be customized by sending configuration data to `msg.ui_control.tabulator`.
117 
118 ![customized table](https://raw.githubusercontent.com/node-red/node-red-ui-nodes/master/node-red-node-ui-table//ui-table-custom.png)
119 
120 by adding ***headers***, ***footers***, ***line*** or ***column grouping*** it is sometimes not possible to determine the amount of lines. Therefore the height can be defined by sending `msg.ui_control.customHeight=lines`.
121 
122 Example flow "3 ui_control table.json" file can be found in the examples folder
123 
124 - grouped columns by nesting column definition in ` ui_control.tabulator.columns`
125 - first column ```frozen``` from horizontal scrolling
126 - `formatterParams` to define min/max, color, legend or other parameters for `progress` and `planText` formatters
127 - functions to format legend values
128 ``` javascript
129 // add a unit
130 function(cell, formatterParams, onRendered){
131  return cell.getValue()+"Â°C";
132 }
133 ```
134 or more sophisticated using html
135 ``` javascript
136 // convert Number to Icons
137 function(cell, formatterParams, onRendered){
138  var html="<i class=\"";
139  switch(cell.getValue()) {
140  case 0: html+="fa fa-calendar-check-o"; break;
141  case 1: html+="fa fa-hand-o-up"; break;
142  case 2: html+="fa fa-suitcase"; break;
143  case 3: html+="fa fa-spinner fa-spin fa-fw"; break;
144  }
145  html+='\"></i>';
146  return html;
147 }
148 ```
149 - `topCalc` for average and min/max calculations
150 - custom icons for `tickCross` formatter
151 - `tick` formatter
152 - `groupBy` parameter to use group lines. `groupHeader` function to format legend and adding html tags (Insert a field name in the groupBy paramter at the end of json in the change node to use this feature)
153 - `columnResized` callback function to receive a message when the user resize a column
154 ``` javascript
155 function(column){
156  var newColumn = {
157  field: column._column.field,
158  visible: column._column.visible,
159  width: column._column.width,
160  widthFixed: column._column.widthFixed,
161  widthStyled: column._column.widthStyled
162  };
163  this.send({
164  ui_control:{callback:'columnResized',columnWidths:newColumn}
165  });
166 }
167 ```
168 - use `this.send({})` to pass result to Node-RED. (to avoid a loopback add`ui_control.callback="someText"`)
169 ```javascript
170  this.send({topic: "anyTopic",payload:"anyPayload",ui_control: {callback:"myCallback"}});
171 ```
172 - all parameters are named according to tabulator documentation. Use ```field``` instead of ```Property``` used in node configuration
173 - no validation of `msg.ui_control` data is performed! So if you don`t get the results you expect take a look on your browsers console.
